### 2.6.4 AP Form Stock

The **AP Form Stock** page provides a read-only stock monitoring view for the Allocation Planning module. For a selected Year, Week, and Source DC, it displays the current On Hand Stock (OHS) per FA Brand alongside the projected remaining stock for each day of the week after planned allocations are deducted.

This page allows the planning team to quickly identify which FA codes are at risk of running out of stock within the week ŌĆö either due to high allocation volumes or a low opening balance ŌĆö without opening the full Picking List detail. Negative daily stock values are colour-coded for immediate visual identification.

#### **3.2. Daily Stock Calculation**

Each day column (Monday to Sunday) is computed as:

```
Stock[day] = OHS ŌłÆ SUM(APLPlanAllocation.QtyBox WHERE DispatchDate = WeekDay[day])
```

Where:
- `OHS` = On Hand Stock from `APLStockDC` for the FA code, Source DC, Year, and Week.
- `APLPlanAllocation.QtyBox` = Sum of all allocation boxes planned for that FA code on the given dispatch day, within the same Year, Week, and Source DC.
- `DispatchDate` = date column on `APLPlanAllocation` recording when goods physically move out of the Source DC.

The deduction is **per-day** (not cumulative) ŌĆö each day column independently shows OHS minus that day's total outbound allocation. This makes the column represent the remaining DC stock after all planned dispatches for that day have left the warehouse.

### **5. Key Query**

```sql
-- OHS and daily remaining stock per FA Brand
SELECT
    S.FaCode,
    S.BrandCode,
    S.StockBox                          AS OHS,
    SUM(CASE WHEN D.DayOfWeek = 1 THEN A.QtyBox ELSE 0 END) AS AllocMon,
    SUM(CASE WHEN D.DayOfWeek = 2 THEN A.QtyBox ELSE 0 END) AS AllocTue,
    SUM(CASE WHEN D.DayOfWeek = 3 THEN A.QtyBox ELSE 0 END) AS AllocWed,
    SUM(CASE WHEN D.DayOfWeek = 4 THEN A.QtyBox ELSE 0 END) AS AllocThu,
    SUM(CASE WHEN D.DayOfWeek = 5 THEN A.QtyBox ELSE 0 END) AS AllocFri,
    SUM(CASE WHEN D.DayOfWeek = 6 THEN A.QtyBox ELSE 0 END) AS AllocSat,
    SUM(CASE WHEN D.DayOfWeek = 7 THEN A.QtyBox ELSE 0 END) AS AllocSun
FROM APLStockDC S
LEFT JOIN APLPlanHeader     H ON H.SourceCode = S.Plant
                              AND H.Year       = S.Year
                              AND H.Week       = S.Week
LEFT JOIN APLPlanDay        D ON D.PlanHeaderId = H.Id
LEFT JOIN APLPlanTrip       T ON T.PlanDayId    = D.Id
LEFT JOIN APLPlanAllocation A ON A.PlanTripId   = T.Id
                              AND A.FaCode      = S.FaCode
WHERE S.Plant = @Source
  AND S.Year  = @Year
  AND S.Week  = @Week
GROUP BY S.FaCode, S.BrandCode, S.StockBox
ORDER BY S.BrandCode, S.FaCode
```

The daily remaining stock columns are then computed in the application layer:

```
Stock[Mon] = OHS ŌłÆ AllocMon
Stock[Tue] = OHS ŌłÆ AllocTue
...
Stock[Sun] = OHS ŌłÆ AllocSun
```

### **6. Export Excel**

The Export Excel button triggers a server-side download.

**Endpoint:** `GET /APFormStock/ExportExcel`

**Query parameters:**

| Parameter | Description |
| :--- | :--- |
| `year` | Selected planning year |
| `week` | Selected planning week |
| `source` | Source DC plant code (ZD4A or ZD7J) |
| `negOnly` | `1` if Negative Stock Only checkbox is checked; `0` otherwise |

**Output columns:** Brand, FABrand, OHS, Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday. Negative stock cells retain colour coding in the exported file.

### **7. UI Behaviour**

- **FABrand link** ŌĆö clicking navigates to the AP Planning Book detail for that FA code and source, allowing the planner to review or edit the underlying allocation plan.
- **Negative Stock Only filter** ŌĆö when checked, only rows where at least one of the seven day columns is negative are shown. Rows where all daily values are Ōēź 0 are hidden.
- **Keyword search** ŌĆö filters by Brand code OR FABrand code. Partial matches are supported (case-insensitive). Applied client-side, resets pagination to page 1.
- **Page size change** ŌĆö resets to page 1 and re-renders immediately.
- The page is **read-only** ŌĆö no allocation edits are made here.
